# Logger üìä

AzuraJS integrates a contextual logging system that adapts to the execution environment. It provides detailed debugging information during development while offering concise, performance-focused logs in production.

## Logger Configuration

The logger's behavior is controlled via the `logging` object in your `azura.config.ts` configuration file.

```typescript
// azura.config.ts
import type { ConfigTypes } from "azurajs/config";

const config: ConfigTypes = {
  environment: "development",
  logging: {
    enabled: true,
    showDetails: true,
  },
};

export default config;
```

### `logging` Options

| Property      | Type      | Default      | Description                                                              |
| :------------ | :-------- | :----------- | :----------------------------------------------------------------------- |
| `enabled`     | `boolean` | `true`       | Activates or deactivates the logging middleware.                         |
| `showDetails` | `boolean` | `true` (dev) | If enabled, includes request details like query params and body content. |

## Request Logs

When active, the system automatically records incoming requests and outgoing responses, capturing HTTP methods, URLs, response times, and status codes.

## Using the Internal Logger

You can utilize the core `logger` function to create custom log entries within your application logic. These logs integrate directly with the framework's logging system.

```typescript
import { logger } from "azurajs/utils/logger";

// Standard usage
logger("info", "User successfully logged in!");
logger("warn", "Unauthorized access attempt.");
logger("error", "Error processing user data.", { userId: 123 });
```

### Supported Log Levels

- `"info"`
- `"warn"`
- `"error"`
- `"debug"`
