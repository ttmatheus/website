# Swagger / OpenAPI üìñ

Documenting your API is essential for collaboration between developers. AzuraJS facilitates this process with native integration that generates OpenAPI 3.0 specifications and an interactive visual interface (Swagger UI).

## Quick Setup ‚ö°

To enable Swagger, use the `setupSwaggerWithControllers` utility function in your main file:

```typescript
import { AzuraClient, setupSwaggerWithControllers } from "azurajs";
import { UserController } from "./controllers/UserController";

const app = new AzuraClient();

setupSwaggerWithControllers(app, {
  title: "My Azura API",
  version: "1.0.0",
  description: "Automatic route documentation",
  path: "/docs" // Path to access the UI
}, [UserController]);

await app.listen(3000);
```

Now, by accessing `http://localhost:3000/docs`, you will see the Swagger interface ready for use!

## Swagger Decorators üè∑Ô∏è

You can enrich the documentation using specific decorators in your controllers and methods:

### `@ApiTags(...tags)`
Groups endpoints by categories in the interface.
```typescript
@ApiTags("Users")
@Controller("/users")
class UserController { ... }
```

### `@Swagger(config)`
Defines the summary and detailed description of an endpoint.
```typescript
@Swagger({
  summary: "Get profile",
  description: "Returns private data of the authenticated user"
})
@Get("/profile")
getProfile() { ... }
```

### `@ApiResponse(status, description, options?)`
Documents possible route returns.
```typescript
@ApiResponse(200, "Success", { type: UserSchema })
@ApiResponse(401, "Unauthorized")
@Get()
list() { ... }
```

## JSON Specification üìÑ

AzuraJS also automatically exposes the specification JSON file at the `/api-spec.json` path. This allows you to import your API into tools like Postman, Insomnia, or generate clients automatically.
