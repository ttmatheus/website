# Swagger / OpenAPI üìñ

Effective API documentation is essential for developer collaboration. AzuraJS simplifies this with native tools that automatically generate OpenAPI 3.0 specifications and serve an interactive Swagger UI.

## Quick Setup ‚ö°

Enable Swagger by invoking `setupSwaggerWithControllers` within your application entry point.

```typescript
import { AzuraClient, setupSwaggerWithControllers } from "azurajs";
import { UserController } from "./controllers/UserController";

const app = new AzuraClient();

setupSwaggerWithControllers(
  app,
  {
    title: "My Azura API",
    version: "1.0.0",
    description: "Automatic route documentation",
    path: "/docs",
  },
  [UserController],
);

await app.listen(3000);
```

Access the documentation at `http://localhost:3000/docs`.

## Swagger Decorators üè∑Ô∏è

Enhance documentation detail by decorating controllers and methods.

### `@ApiTags(...tags)`

Organize endpoints into categories.

```typescript
@ApiTags("Users")
@Controller("/users")
class UserController { ... }
```

### `@Swagger(config)`

Provide endpoint summaries and descriptions.

```typescript
@Swagger({
  summary: "Get profile",
  description: "Returns private data of the authenticated user"
})
@Get("/profile")
getProfile() { ... }
```

### `@ApiResponse(status, description, options?)`

Document potential responses and schemas.

```typescript
@ApiResponse(200, "Success", { type: UserSchema })
@ApiResponse(401, "Unauthorized")
@Get()
list() { ... }
```

## JSON Specification üìÑ

AzuraJS automatically serves the raw OpenAPI JSON specification at `/api-spec.json`. This endpoint can be used to import your API definition into tools like Postman or Insomnia, or for client generation.
