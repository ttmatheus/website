# Rate Limiting üõ°Ô∏è

AzuraJS offers an integrated Rate Limiting middleware that helps protect your API against brute-force attacks, abuse, and denial-of-service (DDoS) by limiting the number of requests a client can make within a given period.

## Global Configuration

You can configure Rate Limiting globally through your `azura.config.ts` file:

```typescript
// azura.config.ts
import type { ConfigTypes } from "azurajs/config";

const config: ConfigTypes = {
  // ...
  plugins: {
    rateLimit: {
      enabled: true,           // Enable Rate Limiting
      limit: 100,              // Maximum of 100 requests
      timeframe: 60000,        // Per minute (60000ms)
      // message: "Too many requests, please try again later.", // Custom message
      // trustProxy: true,      // If behind a reverse proxy
    },
  },
};

export default config;
```

### `rateLimit` Options

| Property | Type | Description |
| :--- | :--- | :--- |
| `enabled` | `boolean` | Enables or disables the Rate Limiting middleware. Default: `false`. |
| `limit` | `number` | The maximum number of requests allowed within the `timeframe`. Default: `60`. |
| `timeframe` | `number` | The time window in milliseconds for which the `limit` applies. Default: `60000` (1 minute). |
| `message` | `string` | Custom error message to be sent when the limit is exceeded. Default: `"Too many requests, please try again later."`. |
| `trustProxy` | `boolean` | If `true`, the client IP will be resolved from `X-Forwarded-For` headers. Default: `false`. |

## Per Route/Controller Rate Limiting

To apply Rate Limiting to specific controllers or routes, you can use the `@UseMiddleware` decorator with AzuraJS's `rateLimit` function.

```typescript
import { Controller, Get, UseMiddleware } from "azurajs/decorators";
import { rateLimit } from "azurajs/plugins";

const tenRequestsPerMinute = rateLimit(10, 60 * 1000);

@Controller("/auth")
export class AuthController {
  @UseMiddleware(tenRequestsPerMinute)
  @Get("/login")
  login() {
    return "Login Page";
  }

  @UseMiddleware(rateLimit(5, 30 * 1000)) // 5 requests every 30 seconds
  @Post("/register")
  register() {
    return "User Registration";
  }
}
```

## How It Works

The Rate Limiting middleware tracks the number of requests from each IP (or identifier, if `trustProxy` is enabled) within the defined time window. If the limit is exceeded, it returns a `429 Too Many Requests` status.
