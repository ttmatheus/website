# Validation âœ…

AzuraJS includes a robust, built-in validation system designed to ensure data integrity before it reaches your business logic. It prevents invalid data processing and provides automatic type inference for TypeScript users.

The validator offers a fluent, chainable API inspired by Zod, allowing for concise schema definitions and rigorous data checking.

### Schema Definition

Define schemas using the `v` utility to create validation rules.

```typescript
import { v } from "azurajs/validators";

const CreateUserSchema = v.object({
  username: v.string().min(3).max(20),
  email: v.string().email(),
  age: v.number().int().min(18).optional(),
});

type CreateUserDto = v.infer<typeof CreateUserSchema>;
```

## Validating in Controller ğŸ¯

Invoke the schema's `.safeParse()` method within your controller to validate incoming request bodies.

```typescript
import { Controller, Post, Body, Res } from "azurajs/decorators";
import type { ResponseServer } from "azurajs/types";

@Post()
async createUser(@Body() body: unknown, @Res() res: ResponseServer) {
  const result = CreateUserSchema.safeParse(body);

  if (!result.success) {
    return res.status(400).json({
      error: "Validation failed",
      details: result.error.format()
    });
  }

  const data = result.data;
  // Proceed with safe, typed execution
}
```

## Supported Data Types ğŸ“Š

The validator supports a comprehensive set of data types and constraints:

- `v.string()`: String validation with helpers like `.email()`, `.url()`, `.uuid()`, `.min()`, `.max()`.
- `v.number()`: Numeric validation with helpers like `.int()`, `.positive()`, `.min()`, `.max()`.
- `v.boolean()`: Validates boolean values.
- `v.array(type)`: Validates arrays of a specific type.
- `v.object(shape)`: Validates nested object structures.
- `v.enum([values])`: Validates against a set of allowed values.
