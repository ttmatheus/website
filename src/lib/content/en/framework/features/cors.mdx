# CORS (Cross-Origin Resource Sharing) üåê

CORS is a security mechanism that allows or restricts resource requests from external domains. AzuraJS provides a flexible, production-ready CORS plugin that supports both global and per-route configuration.

## Global Configuration

Enable and customize CORS via the `plugins` object in `azura.config.ts`.

```typescript
// azura.config.ts
import type { ConfigTypes } from "azurajs/config";

const config: ConfigTypes = {
  plugins: {
    cors: {
      enabled: true,
      origins: ["http://localhost:3000", "https://yourwebsite.com"],
      methods: ["GET", "POST", "PUT", "DELETE", "PATCH"],
      allowedHeaders: ["Content-Type", "Authorization"],
      exposedHeaders: ["X-Custom-Header"],
      credentials: true,
      maxAge: 3600,
    },
  },
};

export default config;
```

### `cors` Options

| Property         | Type       | Description                                                                           |
| :--------------- | :--------- | :------------------------------------------------------------------------------------ |
| `enabled`        | `boolean`  | Activates the CORS middleware. Default: `false`.                                      |
| `origins`        | `string[]` | List of allowed origins. Use `["*"]` to allow all (unsafe for production).            |
| `methods`        | `string[]` | Allowed HTTP methods. Default includes standard verbs.                                |
| `allowedHeaders` | `string[]` | Allowed request headers.                                                              |
| `exposedHeaders` | `string[]` | Response headers accessible to the browser.                                           |
| `credentials`    | `boolean`  | Permits cookies and authorization headers in cross-origin requests. Default: `false`. |
| `maxAge`         | `number`   | Preflight cache duration in seconds. Default: `3600`.                                 |

## Preflight Requests

AzuraJS automatically handles `OPTIONS` preflight requests for complex CORS operations, ensuring seamless interaction with browser security policies without additional route configuration.
