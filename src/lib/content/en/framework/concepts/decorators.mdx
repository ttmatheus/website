# Decorators üè∑Ô∏è

Decorators are a powerful TypeScript feature that allows you to add metadata and logic to classes, methods, properties, and parameters. In AzuraJS, decorators form the foundation for defining routes, injecting dependencies, and configuring API behavior declaratively.

## Enabling Decorators

To use decorators, update your `tsconfig.json` with the following compiler options:

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

## Types of Decorators

### 1. Class Decorators

These are applied to classes to define their role (e.g., as a Controller) and apply global configurations.

- `@Controller(prefix?: string)`: Marks a class as a controller. Optionally accepts a route prefix.

```typescript
import { Controller } from "azurajs/decorators";

@Controller("/api/v1")
export class MyController {}
```

### 2. Method Decorators

These decorators map class methods to specific HTTP verbs and route paths.

- `@Get(path?: string)`
- `@Post(path?: string)`
- `@Put(path?: string)`
- `@Delete(path?: string)`
- `@Patch(path?: string)`
- `@Head(path?: string)`
- `@Options(path?: string)`

In the example below, `getUser` maps to `GET /users/:id` and `createUser` maps to `POST /users`:

```typescript
import { Controller, Get, Post } from "azurajs/decorators";

@Controller("/users")
export class UserController {
  @Get("/:id")
  getUser() {
    /* ... */
  }

  @Post()
  createUser() {
    /* ... */
  }
}
```

### 3. Parameter Decorators

Use these to inject request data directly into your method arguments, avoiding manual parsing.

- `@Req()`: Request object
- `@Res()`: Response object
- `@Next()`: Next function (middleware)
- `@Param(name?: string)`: Route parameters
- `@Query(name?: string)`: Query parameters
- `@Body(name?: string)`: Request body
- `@Headers(name?: string)`: Request headers
- `@Cookies(name?: string)`: Request cookies
- `@Ip()`: Client IP

```typescript
import { Controller, Get, Param, Query, Body, Post } from "azurajs/decorators";

@Controller("/products")
export class ProductController {
  @Get("/:id")
  getProduct(
    @Param("id") productId: string,
    @Query("version") version?: string,
  ) {
    return { productId, version };
  }

  @Post()
  createProduct(@Body() productData: any) {
    return { created: productData };
  }
}
```

## Application Registration

Decorators are only processed when registered. You must call `applyDecorators()` in your entry file.

```typescript
import { AzuraClient } from "azurajs";
import { applyDecorators } from "azurajs/decorators";
import { UserController } from "./controllers/UserController";

const app = new AzuraClient();
applyDecorators(app, [UserController]);
```
