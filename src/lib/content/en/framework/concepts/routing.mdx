# Routing ğŸ›£ï¸

The AzuraJS routing system is built on a high-performance **Radix Tree** (Prefix Tree). This ensures that route lookup speed is extremely fast and constant, regardless of how many routes your application has.

## Route Patterns ğŸ§©

Routes are defined in controller methods using decorators.

### Static Routes
Fixed paths that do not change.
```typescript
@Get("/health")
check() { return "OK"; }
```

### Dynamic Routes (Parameters)
Use the `:` prefix to define variables in the URL.
```typescript
@Get("/users/:id")
getUser(@Param("id") id: string) {
  return { id };
}
```

### Multiple Parameters
You can combine multiple parameters in a single route.
```typescript
@Get("/org/:orgId/projects/:projectId")
getProject(@Param("orgId") org: string, @Param("projectId") project: string) {
  return { org, project };
}
```

## Supported HTTP Methods ğŸ“¡

AzuraJS supports all standard methods:

- `@Get(path)`
- `@Post(path)`
- `@Put(path)`
- `@Delete(path)`
- `@Patch(path)`
- `@Head(path)`
- `@Options(path)`

## Route Priority ğŸ”¢

The router follows a logical precedence order:
1.  **Static Routes**: `/users/profile` will always have priority over `/users/:id`.
2.  **Parameterized Routes**: `/users/:id` is checked if no static route matches.
3.  **Wildcards**: Routes that catch everything (if implemented).

## Case Sensitivity ğŸ” 

By default, routing in AzuraJS is **case-sensitive** for the path, but you can configure global behavior in `azura.config.ts`.
