# Configuration ‚öôÔ∏è

AzuraJS is highly configurable via the `azura.config.ts` file (also supports `.js`, `.json`, `.yaml`). This file is automatically loaded upon `AzuraClient` initialization, allowing you to tailor the server's behavior, plugins, and environment settings.

## Configuration File Structure

The following example demonstrates a standard configuration without inline comments. Refer to the tables below for detailed explanations of each property.

```typescript
import type { ConfigTypes } from "azurajs/config";

const config: ConfigTypes = {
  environment: "development",
  server: {
    port: 3000,
    ipHost: true,
    https: false,
    cluster: false,
    trustProxy: false,
  },
  logging: {
    enabled: true,
    showDetails: true,
  },
  plugins: {
    cors: {
      enabled: true,
      origins: ["*"],
      methods: ["GET", "POST", "PUT", "DELETE", "PATCH"],
      allowedHeaders: ["Content-Type", "Authorization"],
    },
    rateLimit: {
      enabled: false,
      limit: 100,
      timeframe: 60000,
    },
  },
};

export default config;
```

## Detailed Configuration Options

### Environment

The `environment` property defines the execution mode of the application. It affects default logging behaviors and performance optimizations.

| Value           | Description                                          |
| :-------------- | :--------------------------------------------------- |
| `"development"` | Enables detailed logging and debug features.         |
| `"production"`  | Optimizes for performance and reduces log verbosity. |
| `"test"`        | Configures the environment for automated testing.    |

### Server Configuration

These settings control the core HTTP/HTTPS server behavior.

| Property     | Type                            | Default | Description                                                                                      |
| :----------- | :------------------------------ | :------ | :----------------------------------------------------------------------------------------------- |
| `port`       | `number`                        | `3000`  | The port the server will listen on.                                                              |
| `ipHost`     | `boolean`                       | `true`  | Attempts to resolve the host IP automatically.                                                   |
| `https`      | `boolean`                       | `false` | Enables HTTPS support. Requires `key` and `cert`.                                                |
| `key`        | `string`                        | -       | Path to the SSL private key file.                                                                |
| `cert`       | `string`                        | -       | Path to the SSL certificate file.                                                                |
| `cluster`    | `boolean`                       | `false` | Enables experimental cluster mode to utilize multiple CPU cores.                                 |
| `trustProxy` | `boolean \| string \| string[]` | `false` | Configures trust for reverse proxies (e.g., Nginx, Cloudflare) to correctly identify client IPs. |

### Logging Configuration

Controls the internal logging system.

| Property      | Type      | Default      | Description                                                                                    |
| :------------ | :-------- | :----------- | :--------------------------------------------------------------------------------------------- |
| `enabled`     | `boolean` | `true`       | Toggles the logging system on or off.                                                          |
| `showDetails` | `boolean` | `true` (dev) | If true, logs include detailed information like query parameters, request bodies, and headers. |

### Plugins

Configure built-in plugins such as CORS and Rate Limiting directly in the config file.

#### CORS

| Property         | Type       | Description                                                                     |
| :--------------- | :--------- | :------------------------------------------------------------------------------ |
| `enabled`        | `boolean`  | Toggles CORS support.                                                           |
| `origins`        | `string[]` | List of allowed origins (e.g., `["http://localhost:3000"]`) or `["*"]` for all. |
| `methods`        | `string[]` | List of allowed HTTP methods.                                                   |
| `allowedHeaders` | `string[]` | List of allowed HTTP headers.                                                   |

#### Rate Limit

| Property    | Type      | Description                                               |
| :---------- | :-------- | :-------------------------------------------------------- |
| `enabled`   | `boolean` | Toggles Rate Limiting.                                    |
| `limit`     | `number`  | Maximum number of requests allowed within the timeframe.  |
| `timeframe` | `number`  | Time window in milliseconds (e.g., `60000` for 1 minute). |

## Environment Variables

AzuraJS supports automatic loading of environment variables from a `.env` file located in the project root.

**Example `.env` file:**

```text
PORT=8080
DATABASE_URL=postgres://user:pass@host:port/db
```

These variables are accessible via `process.env`. You can reference them within your `azura.config.ts`:

```typescript
const config: ConfigTypes = {
  server: {
    port: Number(process.env.PORT) || 3000,
  },
  // ...
};
```
